## 一、项目概述

### 是什么

**RL Odyssey（强化学习奥德赛）** 是一款交互式、游戏化的 Web 教育应用，通过像素风航海冒险的方式，向初学者（目标受众：初中生，约 14-15 岁）讲授强化学习（Reinforcement Learning）的核心算法。

核心理念：**"如果一个初中生在 2 分钟内看不懂，就重写。"**

玩家扮演一名在**智慧之海（Sea of Intelligence）**中航行的探险家，前往不同的岛屿链，每个港口（关卡）介绍一种 RL 算法。所有概念均通过类比、可视化和动手交互传授，绝不使用晦涩的学术术语。

### 目标受众

- **主要受众**：初中生（14-15 岁），无 ML / 编程基础
- **次要受众**：RL 入门学习者，想通过可视化方式理解算法原理的开发者

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| 样式 | Tailwind CSS + 自定义像素风 CSS |
| 状态管理 | Zustand（含 localStorage 持久化） |
| 路由 | React Router v6 |
| 可视化 | HTML5 Canvas + SVG |
| 国际化 | react-i18next（中/英双语） |
| 神经网络 | 手写 SimpleNN（TypeScript，无 ML 库依赖） |
| 部署 | Vercel |

**重要约束**：纯前端应用，无后端服务器，所有 RL 计算均在浏览器中运行。

## 三、已实现的关卡（含玩法详述）

### 第 0 章：起航港教程——训练你的像素宠物

**状态**：✅ 已完成

**教学目标**：在进入任何算法关卡前，玩家先体验完整的 RL 核心循环（状态→动作→奖励→学习），不涉及任何算法术语。

**玩法流程（约 5 分钟）**：

**步骤 1：宠物选择**
玩家从 4 种像素宠物中选择一个（🦜 鹦鹉 / 🦁 狮子 / 🐙 章鱼 / 🐢 乌龟），作为整个游戏的专属角色。

**步骤 2：手动控制（1 分钟）**
4×4 格子世界中，宠物出现在起点，奖励藏在某个格子。玩家通过方向键或点击操控宠物找到奖励，直接体验"探索"的感觉。

**步骤 3：蒙眼挑战（1 分钟）**
画面变暗（迷雾遮盖），玩家必须凭记忆找到奖励。引出问题："如果能让宠物自己学会哪个方向好，岂不是更好？"

**步骤 4：学习循环演示（2 分钟）**
用简单标签（无术语）可视化完整 RL 循环：
- 🐕 **你**（Agent）→ 看周围（State）→ 选方向（Action）→ 得到反馈（Reward）→ 记住（Learn）→ 重复

玩家观看宠物自动学习，画面上出现"记忆热图"，逐步填满，直观展示学习进展。

**步骤 5：大问题（1 分钟）**
提出总结："有很多方法教宠物学习，每座岛屿教一种不同的方法。你要先去哪里？"
世界地图开放，进入航线选择。

#### 港口 1：土匪港（Port Bandit）——多臂土匪算法

**状态**：✅ 已完成（8 步完整流程）

**对应算法**：Multi-Armed Bandit（多臂老虎机）
**核心概念**：探索与利用（Exploration vs Exploitation）、ε-贪心策略、UCB 置信上界

**玩法详述**：

**感受问题（Feel the Problem）**
画面呈现 5 个藏宝箱，每个箱子有未知的随机奖励分布。玩家有 20 次机会手动拉箱子，体验"多试新箱子 vs 坚守已知最好箱子"的两难困境。金币粒子从箱子飞出，视觉直观。

**认识算法（Meet the Algorithm）**
分步动画解释两种策略：
- **ε-贪心**：大多数时候选最好的，偶尔随机尝试新的（ε 控制好奇心比例）
- **UCB（置信上界）**：优先选"被尝试次数少但可能很好"的选项（探索不确定性）

**观察学习（Watch It Learn）**
算法自动运行，实时折线图显示：累计奖励曲线、各臂被选次数。玩家可用滑块调整 ε 值或切换为 UCB，对比效果差异。

**任务挑战——赌徒困境（Quest: The Gambler's Dilemma）**
100 回合内，5 个宝箱，最大化金币收入。第 50 回合时宝箱奖励分布悄悄改变（非稳态老虎机），测试算法的适应能力。

| 评分 | 条件 | 金币奖励 |
|------|------|---------|
| C（新手）| 超过纯随机策略 | 150 |
| B（标准）| 1.5× 随机收入 | 300 |
| A（优秀）| 2.0× 随机收入 | 600 |
| S（传奇）| 2.5× 随机收入 | 900 |

**卡牌奖励**：「探险家的直觉」（Explorer's Instinct）
完成后解锁：港口 Q-Table

#### 港口 2：Q 表格港（Port Q-Table）——Q-Learning

**状态**：✅ 已完成（8 步完整流程）

**对应算法**：Q-Learning（Q 学习）
**核心概念**：Q 值、贝尔曼方程（"向未来的自己借建议"）、Q 表格（状态-动作价值表）

**玩法详述**：

**感受问题**
二维格子世界地图，有陷阱、宝藏、出口。玩家手动操控角色探索 20 步，亲身感受"哪些路线危险、哪些路线有价值"。

**认识算法**
类比解释："想象你第一天到新学校，每天尝试不同路线去食堂。发现周二走左边走廊能到最好的位置，你心里就记下来：'左边=好'。这张心理地图就是 Q 表格。"

配合动画展示贝尔曼方程的直觉：`Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]`，每项都用中文白话解释。

**观察学习**
Q-Learning 自动训练，格子上叠加热力图实时更新（蓝→红，表示 Q 值低→高）。玩家可见每一步的 Q 值变化，可调学习率（α）、折扣因子（γ）、探索率（ε）。

**任务挑战——逃离诅咒之岛（Quest: Escape the Cursed Island）**
玩家先手动走一遍迷宫，记录步数。然后训练 Q-Learning 算法，若找到比玩家更短的路径则通过。

| 评分 | 条件 | 金币奖励 |
|------|------|---------|
| C | 与玩家相当 | 250 |
| B | 比玩家短 20% | 500 |
| A | 比玩家短 40% | 1000 |
| S | 找到最优路径 | 1500 |

**卡牌奖励**：「地图绘制者」（The Cartographer）
完成后解锁：港口 Deep（DQN）

#### 港口 3：深度港（Port Deep）——DQN 深度 Q 网络

**状态**：✅ 已完成（8 步完整流程）

**对应算法**：DQN（Deep Q-Network，深度 Q 网络）
**核心概念**：神经网络作为 Q 函数近似、经验回放（"把记忆卡片打乱学习"）、目标网络（"稳定的老师"）

**玩法详述**：

**感受问题**
像素风 Breakout 游戏（打砖块），玩家手动操控挡板弹球打砖，体验游戏难度，记录自己的得分作为基准。

**认识算法**
解释 Q-Learning 的局限："格子太多了，Q 表格会大到放不下！"
引入神经网络作为"价值估算助手"：输入游戏状态（球位置/速度/挡板位置），输出每个动作的 Q 值估算。

重点解释三大 DQN 技巧：
- **经验回放**："就像学习时打乱闪卡的顺序，让 AI 不只记住最近发生的事"
- **目标网络**："有一个单独的、慢速更新的'老师'网络，让训练更稳定"
- **ε-衰减**："从好奇探索逐渐转变为利用所学"

**观察学习**
DQN 自动学习打砖块，实时显示：
- Breakout 游戏画面（含 Q 值叠加可视化）
- 训练损失曲线（Loss）
- 累计奖励曲线
- 可调参数：学习率、批次大小、经验回放缓冲区大小

**任务挑战——像素街机挑战（Quest: Pixel Arcade Challenge）**
训练 DQN 智能体，目标是超越玩家之前设定的手动基准分数。

| 评分 | 条件 | 金币奖励 |
|------|------|---------|
| C | 与玩家得分持平 | 400 |
| B | 1.2× 玩家得分 | 800 |
| A | 1.5× 玩家得分 | 1200 |
| S | 2.0× 玩家得分 | 2400 |

**卡牌奖励**：「神经导航者」（Neural Navigator）
价值群岛第一条主线完成。

### 核心数据流

```
Zustand Store (全局状态)
    ↑ dispatch                    ↓ read
React 组件 ←──── props ────→ Canvas 渲染器
    ↑                              ↓ 只读
用户交互（滑块/按钮/键盘）     算法可视化数据
```

Canvas 是单向只读的叶节点，永不直接修改应用状态。

### 算法接口（统一规范）

所有 RL 算法实现统一接口：

```typescript
interface RLAlgorithm<S, A> {
  step(state: S): { action: A; metadata: StepMetadata }
  update(experience: Experience<S, A>): UpdateMetadata
  reset(): void
  getVisualizationData(): VisualizationData
  getHyperparameters(): HyperParameter[]
  setHyperparameter(key: string, value: number): void
}
```

### 港口通用流程（8 步模板）

所有算法关卡复用同一套流程框架：

```
故事介绍 → 概念入门 → 感受问题 → 认识算法
    → 观察学习 → 任务挑战 → 总结奖励 → 解锁下一关
```

每步均为独立 React 组件，按需懒加载，支持保存/恢复进度。
